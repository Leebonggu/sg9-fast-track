통합 관리 시스템 — 접근 방식 3가지

어떤 방향이 맞는지 골라주세요

A

Google Sheets 통합 집계 시트

방법: 소유자 목록을 Sheets로 변환 → Apps Script가 주기적으로 각 시트(사전동의·신통접수·설문)를 읽어 "통합현황" 시트 자동 갱신

결과: 엑셀처럼 생긴 표에서 필터/정렬로 미완료자 확인

장점

  • 개발 거의 없음
  • 추진위원 친숙한 환경
  • Excel 다운로드 바로 가능

단점

  • 수식/Apps Script 복잡
  • 2,830행 실시간 느림
  • 메모 관리 어색
B

⭐ 기존 웹앱(Next.js) 확장 — 추천

방법: 소유자 마스터 시트 추가 → 기존 web/ 앱에 "통합 현황" 페이지 추가. API가 동/호수 기준으로 3개 시스템 join

결과: 웹에서 검색·필터·메모 편집, 미완료자 목록 바로 확인

장점

  • 인프라 이미 있음 (Sheets API, Auth)
  • 검색·필터·메모 쉽게 구현
  • MissingRespondents 컴포넌트 재활용
  • 모바일에서도 접근 가능

단점

  • 개발 공수 1~2일
  • 매번 웹 접속 필요
C

Sheets + 웹앱 하이브리드

방법: Apps Script로 매일 1회 통합 집계 시트 자동 생성 (현황 파악용) + 웹앱은 메모 편집·미완료자 필터만 담당

결과: 시트로 빠르게 현황 보고, 웹앱으로 컨택 관리

장점

  • 역할 분리 명확
  • 시트 공유로 비개발자도 현황 확인

단점

  • 두 시스템 유지 필요
  • 데이터 동기화 주의

추천 이유 (B)

현재 웹앱에 이미 MissingRespondents, SurveyAnalytics, Sheets API 연동이 모두 있습니다. 소유자 마스터 시트 하나 추가하고 join 로직을 붙이면 됩니다. Apps Script 없이 깔끔하게 해결됩니다.
