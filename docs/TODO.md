# 개선사항

## 우선순위 높음

- [ ] **Vercel 배포** — 현재 로컬 개발 중. Vercel에 배포하여 실제 URL로 자원봉사자에게 공유
- [ ] **대시보드 로딩 속도 개선** — 전체현황 시트에 수거완료 수가 없으면 느려질 수 있음. 캐싱 또는 ISR(Incremental Static Regeneration) 적용 검토
- [ ] **v2 onFormSubmit hook 연결** — 구글폼 제출 시 웹 UI에서도 실시간 반영되도록 v2에 hook 1줄 추가 (프로덕션 적용 시)
- [ ] **동별 그리드 로딩 속도** — getBuildingData에서 getRows() 호출이 느림. 필요한 컬럼만 읽도록 최적화

## 우선순위 중간

- [ ] **수거 상태 그리드 범례** — 그리드 하단에 색상 범례 추가 (진한파란=수거완료, 연한파란=전자동의, 연한노란=수동, 회색=빈칸)
- [ ] **동별 그리드에서 접수/수거 소계 분리** — 현재 소계에 접수 수만 표시. 수거완료 수도 표시
- [ ] **검색 기능** — 이름으로 검색하여 어떤 동/호수인지 찾기
- [ ] **새로고침 버튼** — 그리드 화면에서 수동 새로고침 (다른 사람이 입력한 데이터 반영)
- [ ] **입력 시 연락처 필드 추가** — 웹에서 수동 입력 시 연락처도 입력 가능하도록
- [ ] **중복 호수 입력 방지** — 이미 데이터 있는 호수에 addConsent 시 경고

## 우선순위 낮음

- [ ] **PWA (Progressive Web App)** — 홈 화면에 추가 가능하도록, 오프라인 캐시
- [ ] **동별 담당자 배정** — 각 동 담당 자원봉사자 표시
- [ ] **수정 이력 로그** — 누가 언제 어떤 셀을 수정했는지 기록
- [ ] **엑셀 내보내기** — 동별 현황을 엑셀로 다운로드
- [ ] **카카오톡 공유** — 동별 현황 요약을 카카오톡으로 공유
- [ ] **동 구조 관리 UI** — 동별 층/호 구조를 웹에서 수정 가능하도록 (현재는 코드에 하드코딩)
- [ ] **다크모드 지원**
