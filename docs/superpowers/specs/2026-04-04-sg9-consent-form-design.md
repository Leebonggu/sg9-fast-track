# 상계주공 9단지 신속통합기획 사전동의 시스템 설계

## 개요

상계주공 9단지(901~923동, 총 23개동, 2,830세대) 신속통합기획 사전동의를 온라인으로 수집하고, 동별로 관리하기 위한 Google Forms + Google Sheets + Apps Script 기반 시스템.

## 구성 요소

| 구성 | 역할 |
|------|------|
| Google Form | 사전동의 설문 폼 (온라인 제출) |
| Google Sheets | 응답 저장 + 동별 관리 시트 + 전체현황 |
| Apps Script | 동별 자동 분류, 중복 감지, 응답 알림 |

## 제출 대상

- **구분소유자(소유주)만 제출**
- 제출 자체가 동의 의사 표현 (동의/미동의 선택 없음)
- 소유 형태에 따라 3가지로 구분:
  - 단독 소유 + 실거주
  - 공동 소유 + 실거주
  - 단독/공동 소유 + 임대 중 (비거주) → 실거주 주소 추가 입력

## Google Form 구조

### 섹션 1: 설문 안내 (설명 텍스트)

```
[제목]
상계주공 9단지 신속통합기획 사전동의서

[설명]
본 설문은 상계주공 9단지 신속통합기획 추진을 위한 사전동의 확인 목적으로 진행됩니다.

- 제출 대상: 상계주공 9단지 구분소유자 (소유주)
- 본 설문 제출은 신속통합기획 추진에 대한 동의 의사 표현으로 간주됩니다.

[개인정보 처리방침]
1. 수집 항목: 이름, 전화번호, 동/호수, 소유형태, 실거주 주소(해당 시)
2. 수집 목적: 신속통합기획 사전동의 확인 및 동의율 관리
3. 보유 기간: 신속통합기획 절차 완료 시까지
4. 제3자 제공: 신속통합기획 관련 행정기관 제출 목적 외 제공하지 않음
5. 파기: 보유 기간 종료 후 즉시 파기
6. 동의 거부 시: 사전동의 절차에 참여할 수 없습니다.
```

### 섹션 2: 기본 정보

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 이름 | 단답형 | O | |
| 전화번호 | 단답형 | O | 정규식 검증: `^010-?\d{4}-?\d{4}$` |
| 동 | 드롭다운 | O | 901동 ~ 923동 |
| 호수 | 단답형 | O | 직접 입력 (예: 101, 1504) |

### 섹션 3: 소유 형태

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 소유 형태 | 객관식 | O | 아래 4가지 선택지 |
| 실거주 주소 | 단답형 | 조건부 | 임대 중 선택 시에만 표시 |

**소유 형태 선택지:**
1. 단독 소유 + 실거주
2. 공동 소유 + 실거주
3. 단독 소유 + 임대 중 (비거주)
4. 공동 소유 + 임대 중 (비거주)

**조건부 로직:** 3 또는 4 선택 시 → 실거주 주소 입력 섹션으로 이동

## Apps Script 기능

### 1. 동별 자동 분류 (`onFormSubmit`)

- 트리거: 폼 응답 제출 시
- 동작:
  1. 새 응답에서 "동" 값 추출
  2. 해당 동 시트를 찾아서 응답 데이터 복사
  3. 입력경로를 "온라인"으로 설정
  4. 동의서수거여부는 빈 체크박스로 생성

### 2. 중복 제출 감지 (P1)

- 같은 동+호수로 중복 응답이 들어오면:
  - 동별 시트에서 기존 행에 "중복" 표시
  - 최신 응답을 유지하고, 이전 응답은 배경색 변경(회색) + 비고에 "중복(이전 응답)" 표시
  - 관리자에게 중복 알림 발송

### 3. 전화번호 유효성 검증 (P1)

- Google Form 자체 정규식 검증: `^010-?\d{4}-?\d{4}$`
- 허용 형식: `01012345678`, `010-1234-5678`

### 4. 응답 알림 (P1)

- 새 응답 제출 시 지정된 이메일로 알림 발송
- 알림 내용: 이름, 동, 호수, 소유형태, 제출시각
- 일일 요약 알림 (선택): 매일 지정 시각에 당일 응답 건수 요약

## 추후 확장 (P2 — 이후 작업)

### 4. QR코드
- 폼 URL을 QR코드로 생성
- 단지 내 게시판, 엘리베이터 등에 부착

### 5. 진행률 대시보드
- 전체현황 시트에 동별 막대 차트 추가
- 전체 진행률 원형 차트

### 6. 데이터 백업
- Apps Script로 주기적(매일) 시트 자동 백업
- 백업 파일을 Google Drive 특정 폴더에 저장

## 추후 확장 (미정 — 동호수 크롤링)

- 네이버 부동산 API에서 동별 호수 데이터 크롤링
- 각 동별 시트에 전체 호수를 미리 채워놓기
- 응답이 들어오면 해당 호수 행에 자동 매칭
- 미응답 세대를 한눈에 파악 가능
- API endpoint: `https://new.land.naver.com/api/complexes/206/buildings`
  - 현재 rate limit으로 차단됨, 별도 시도 필요

## 구현 순서

### Phase 1 (P1) — 핵심 기능
1. Google Form 생성 (설문 안내 + 기본정보 + 소유형태 + 조건부 로직)
2. Google Sheets 시트 구성 (전체응답 + 전체현황 + 동별 23개 시트)
3. Apps Script: 동별 자동 분류 (`onFormSubmit`)
4. Apps Script: 중복 제출 감지
5. 전화번호 정규식 검증 (폼 설정)
6. Apps Script: 응답 알림 (이메일)

### Phase 2 (P2) — 부가 기능
7. QR코드 생성
8. 전체현황 시트 차트 (진행률 대시보드)
9. 데이터 자동 백업 스크립트

### 미정
10. 네이버 부동산 동호수 크롤링 → 동별 시트 사전 세팅
