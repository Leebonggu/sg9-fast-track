# 상계주공 9단지 사전동의 웹 UI 기획

## 개요

v2 스프레드시트의 동별 리스트 데이터를 **모바일 친화적 웹 UI**로 제공.
자원봉사자가 현장에서 폰으로 동의 현황 조회 + 입력/수정/삭제 가능.

## 기술 스택

- **Apps Script Web App** — 별도 서버/호스팅 비용 없음
- **HTML + CSS + vanilla JS** — Apps Script의 HtmlService 사용
- **데이터 저장소** — 기존 v2 스프레드시트 (원본 리스트 시트 직접 읽기/쓰기)
- **그리드 시트 불필요** — 웹 UI가 리스트 데이터를 그리드로 렌더링

## 데이터 흐름

### 읽기 (그리드 표시)
```
웹 UI → google.script.run.getBuildingData(동)
     → Apps Script: 해당 동 리스트 시트 읽기
     → 호수별로 최신 데이터 정리 (중복 제거)
     → MASTER_CONFIG의 층/호 구조에 맞춰 그리드 데이터 구성
     → JSON 반환 → 웹 UI 렌더링
```

### 쓰기 (이름 입력)
```
웹 UI 셀 터치 → 이름 입력 → 저장
     → google.script.run.addConsent(동, 호수, 이름)
     → Apps Script: 해당 동 리스트 시트에 행 추가
       (입력경로: "수동입력(웹)")
     → 성공 → 웹 UI 셀 즉시 갱신
```

### 수정
```
웹 UI 기존 셀 터치 → 이름 수정 → 저장
     → google.script.run.updateConsent(동, 호수, 새이름)
     → Apps Script: 해당 행 찾아서 성명 업데이트
     → 성공 → 웹 UI 셀 즉시 갱신
```

### 삭제
```
웹 UI 셀 터치 → 삭제 버튼
     → google.script.run.deleteConsent(동, 호수)
     → Apps Script: 해당 행 비고에 "삭제됨" 표시 + 배경 회색
       (행 자체는 삭제 안 함 — 기록 보존)
     → 성공 → 웹 UI 셀 비움
```

## Apps Script 함수 (서버 사이드)

| 함수 | 역할 |
|------|------|
| `doGet(e)` | 웹앱 진입점, HTML 반환 |
| `verifyPassword(pw)` | 비밀번호 확인 |
| `getDashboardData()` | 전체현황 데이터 반환 |
| `getBuildingData(building)` | 동별 그리드 데이터 반환 |
| `addConsent(building, unit, name)` | 수동 동의 추가 |
| `updateConsent(building, unit, name)` | 동의 정보 수정 |
| `deleteConsent(building, unit)` | 동의 삭제 (소프트 삭제) |

## 파일 구조

```
src/
  setup_v2.gs          ← 기존 (수정 없음)
  setup_v2_master.gs   ← 그리드 시트용 (제거 가능)
  webapp.gs            ← 웹앱 서버 로직 (신규)
  webapp.html          ← 웹앱 UI (신규, HTML+CSS+JS 올인원)
```

## 모바일 최적화

- 뷰포트 meta 태그: `width=device-width, initial-scale=1`
- 셀 최소 크기: 44x44px (터치 타겟)
- 그리드 좌우 스크롤 (`overflow-x: auto`)
- 층 라벨 열 고정 (`position: sticky`)
- 모달은 하단에서 올라오는 시트 스타일
- 폰트 크기: 본문 14px, 셀 12px

## 성능 고려

- 동별 데이터 로딩: 한 동 최대 ~120행 → 빠름
- 대시보드: 23개 동 집계 → 1회 호출로 처리
- 캐싱: 동 데이터 로드 후 메모리 캐시, 수정 시만 재로드
- Apps Script 실행 시간: 개별 호출 1-2초 이내

## 제약 사항

- Apps Script Web App은 커스텀 도메인 불가 (script.google.com URL)
- 동시 편집 충돌: last-write-wins (실무상 문제없음)
- Apps Script 일일 실행 한도: 무료 계정 기준 충분
