# recon-sim — 재건축 분담금 시뮬레이터

## 이 프로젝트의 작동 방식

**목적**: 재건축 사업의 조합원 추정분담금을 시뮬레이션. 추진위 발표용 + 주민 개인 확인용. Phase 1은 상계주공9단지(SG9) 고정, Phase 2는 `/[slug]/simulator`로 범용화.

**구조**:
- `lib/calculator.ts` — 모든 계산 공식 집중. 수정 시 이 파일 하나만 건드리면 됨
- `lib/types.ts` — 도메인 타입
- `lib/defaults.ts` — SG9 기본값
- `app/setup`, `app/simulator`, `app/scenario`, `app/slides` — 페이지
- `tests/regression.ts` — 중계주공5단지 PDF 회귀 테스트 (25개 항목)

## 행동 규칙 (Harness)

### 수치 모델 수정 시 (필수)
`lib/calculator.ts`, `lib/types.ts`, `lib/defaults.ts` 수정 → 반드시 `npm test` 실행 → 25/25 pass 확인 후 commit.

실패하면:
1. 의도한 모델 변경인지 확인 (테스트 기대값도 같이 수정 필요한가?)
2. 단순 회귀인지 (코드 버그) 확인
3. PDF 원문(p.21~38) 대조

### 외부 자료는 출처로 검증 후 반영
다른 AI 리뷰, 블로그, 검색 결과는 **주장이지 사실이 아님**. 반영 전:
1. 법령 원문(도정법·재초환법·시행령) 또는 PDF(중계5단지 등) 원문 확인
2. 검증 못 하면 UI/주석에 "확인 필요" 명시, "검증됨"으로 거짓 분류 금지
3. "이미 파라미터화 됨" ≠ "모델이 옳음" — 입력 가능과 계산 정확성은 별개

### 회귀 테스트 기대값 수정 시 주의
`tests/regression.ts`의 `EXPECTED` 값을 바꾸는 건 "PDF 원문과 다르게 가겠다"는 선언. 단순히 테스트 통과시키려고 변경하지 말 것. PDF p.XX에 근거 있어야 함.

### 모델 한계 (자동화 금지)
- FAR 시나리오에서 법적상한 초과 임대 의무 자동 분배: 법령·지역별 차이 커서 보류. 사용자 수동 입력 유지
- 임대 세대수 증가 시 총 연면적 자동 증가: 전용/공급/연면적 관계 가정이 부정확 → 경고만 표시, 자동화 금지

## 단위 규칙

- 내부 계산 단위: **천원** (千원)
- 입력 UI: 만원(시세), 만원/평(평당가), 천원(지출)
- `lib/calculator.ts`의 `manToChun`, `eokToChun`, `chunToEok` 헬퍼 사용

## 검증된 default 값 (중계5단지 PDF로 교차 확인)

| 값 | default | PDF 실측 |
|----|---------|---------|
| 지하 연면적 비율 | 0.4 | 0.4088 |
| 조합원 할인율 | 0.9 | 0.8998 |
| 임대 매각가 비율 | 0.25 | 0.2477 (25평) |
| 평당 공사비 | 800만원 | 조합 800 / 신탁 770 |
| 공사비/총지출 | ~85% | 84.86% |

## 미검증 영역 (기본값만 제공, 실제 적용 전 확인 필요)

- 재초환 2024.3 개정 구간 (시행령 원문 미확인)
- 정상주택가격상승률 정의 (법령은 `max(정기예금이자율, 지역평균상승률)` 의무)

## 실행

```bash
npm run dev     # 개발 서버
npm run build   # 프로덕션 빌드
npm test        # PDF 회귀 테스트 (calculator 수정 후 필수)
```
